# Available Features

Endpoint di bawah ini dipilih karena ringan, aman untuk user, dan tidak membuka akses privileged atau data operasional sensitif.

## Endpoint public aman

Endpoint ini tidak memerlukan User API Token:

| Endpoint | Fungsi |
|---|---|
| `GET /api/v1/health` | Cek status API v1 |
| `GET /api/v1/catalog/robux-packages` | Daftar paket Robux aktif |
| `GET /api/v1/catalog/payment-methods` | Daftar metode pembayaran aktif yang aman ditampilkan |
| `GET /api/v1/articles` | Daftar artikel publik |
| `GET /api/v1/articles/{slug}` | Detail artikel publik |

## Endpoint token utility

| Endpoint | Fungsi |
|---|---|
| `POST /api/v1/auth/token/validate` | Validasi User API Token dan mode token |

## Endpoint user-only

Endpoint ini wajib memakai User API Token dan hanya membaca/memproses data milik pemilik token:

| Endpoint | Fungsi |
|---|---|
| `GET /api/v1/me` | Profil akun pemilik token |
| `GET /api/v1/me/dashboard-summary` | Ringkasan dashboard user |
| `GET /api/v1/me/balance` | Saldo user |
| `GET /api/v1/me/balance/transactions` | Mutasi saldo user |
| `GET /api/v1/me/deposits` | Daftar deposit user |
| `GET /api/v1/me/deposits/{invoice}` | Detail deposit user |
| `POST /api/v1/me/orders` | Buat order dari akun pemilik token |
| `GET /api/v1/me/orders` | Daftar order user |
| `GET /api/v1/me/orders/{invoice}` | Detail order user |
| `GET /api/v1/me/invoices/{invoice}` | Detail invoice user |
| `GET /api/v1/me/invoices/{invoice}/status` | Status invoice user |

<Warning>
Dokumentasi publik ini tidak memuat route operasional, credential provider, callback sistem, maintenance, debug, atau konfigurasi server.
</Warning>
